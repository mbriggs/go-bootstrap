# Agent Config

## Sources

- `ai/common/*.md` feeds the root `CLAUDE.md` / `AGENTS.md`.
- `ai/rules/*.md` defines scoped rules with `claudePaths` frontmatter; they
  land in `.claude/rules/`.
- Nested `CLAUDE.md` files outside `.claude/` are hand-edited directory-scoped
  sources. `bin/sync-agent-config` gives them sibling `AGENTS.md` files.
- Repo-owned skills live under `skills/`. `.agents/skills` and `.claude/skills`
  are relative symlinks managed by `bin/link-skills`.

## Generated files

Don't edit these directly:

- Repo-root `AGENTS.md`, `CLAUDE.md`, `.claude/rules/*.md`.
- Nested `AGENTS.md` files.

Generated files start with the marker:

```md
<!-- Generated by bin/sync-agent-config. Edit ai/ or nested CLAUDE.md files, then re-run. -->
```

If you see that marker, stop and edit the source instead.

## Workflow

After editing sources, run from the repo root:

```sh
bin/sync-agent-config
```

For CI / drift checks:

```sh
bin/sync-agent-config --check
```

Claude's post-edit hook runs sync automatically. The shared stop hook runs
`--check`.

## What Goes Where

- Philosophy and short always-loaded invariants → `ai/common/`.
- A convention's policy and rationale → the skill that owns it,
  `skills/<name>/SKILL.md`.
- Path-scoped pointers into skills → `ai/rules/`.
- The structural *where* (boot, request flow, enforcement map, tooling) →
  `docs/SYSTEM-MAP.md`.
